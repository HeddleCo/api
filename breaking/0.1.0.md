# 0.1.0 breaking-change report

`0.1.0` replaces the unpublished-stability `heddle.v1` generation with the
intentionally incompatible `heddle.api.v1alpha1` package. Field numbering
restarts, services are regrouped into neutral domain interfaces, identifiers
and cursors become typed, stream frames become directional, and dormant RPCs
are removed. The exhaustive disposition is `migration-manifest.json`.

The release requires one coordinated maintenance-window cutover. External old
clients are intentionally unsupported afterward.

Import submission and observation move to the canonical, planned
`OperationService`; the import-specific job and progress messages are not
retained. The exact consumer adoption sequence is recorded in
[`docs/operation-cutover.md`](../docs/operation-cutover.md).

## Handle directory and escrow

The handle feature is retained under `IdentityService`; it is not part of the
dormant-RPC removal. The migration is:

| Legacy RPC | Shared RPC | Verified production evidence |
|---|---|---|
| `heddle.v1.HostedUserService/ClaimHandle` | `heddle.api.v1alpha1.IdentityService/ClaimHandle` | Tapestry caller `src/lib/server/api.ts`; Weft handler `crates/weft-server/src/server/grpc_hosted_impl/user.rs` |
| `heddle.v1.HostedUserService/GetHandleStatus` | `heddle.api.v1alpha1.IdentityService/GetHandleStatus` | Tapestry caller `src/lib/server/api.ts`; Weft handler `crates/weft-server/src/server/grpc_hosted_impl/user.rs` |
| `heddle.v1.HostedUserService/RequestHeldName` | `heddle.api.v1alpha1.IdentityService/RequestHeldName` | Tapestry caller `src/lib/server/api.ts`; Weft handler `crates/weft-server/src/server/grpc_hosted_impl/user.rs` |
| `heddle.v1.HostedUserService/ResolveHandle` | `heddle.api.v1alpha1.IdentityService/ResolveHandle` | Weft handler `crates/weft-server/src/server/grpc_hosted_impl/user.rs`; no Tapestry adapter at this cut |

The shared resolve response uses `HandlePrincipal`, which has no subject field.
Active-name resolution therefore cannot expose the Biscuit subject through this
wire type. A tombstone returns `tombstoned = true` without a principal, while a
never-claimed name returns `NOT_FOUND`. Claim failures that would reveal whether
a name is free, held for somebody else, or active for somebody else share the
same public `NOT_FOUND` shape. Request failures likewise do not distinguish no
hold from a caller-owned hold.

`ClaimHandle` and `RequestHeldName` now require `client_operation_id`. Its
namespace is the authenticated subject plus RPC. An identical normalized-name
retry returns the stored response without repeating the mutation; reusing the
key for another normalized name fails with `FAILED_PRECONDITION`.

The coordinated cutover is blocked on the [Weft shared-service adapter](https://github.com/HeddleCo/weft/issues/591)
and [Tapestry shared-client adapter](https://github.com/HeddleCo/tapestry/issues/163).
HeddleCo/heddle#1021 must not repin until both adapters are ready. There is no
dual registration or legacy alias during the cutover.
