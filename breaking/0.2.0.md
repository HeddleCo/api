# 0.2.0 transport cutover

This release is a coordinated pre-1.0 hard cutover for native Heddle and Weft.

- Rust `client` and `server` features and tonic/tonic-prost generation are
  removed. Consumers dispatch with generated `MethodRoute` and
  `MethodDescriptor` values.
- `CallContext` replaces gRPC metadata for deadlines, bearer capability and
  proof, canonical request proof, human verification, client operation id, and
  trace context.
- `CallFailure` and `CallFailureCode` replace transport status values.
  `StreamFailure.grpc_code` is renamed to `code` with the same stable numeric
  meanings.
- Production hosted-call ALPN is `heddle-api/1`; framing is pinned by
  `tests/fixtures/hosted-call-v1.json`.
- The additive `AgentRun` contract from #35 and `RotateCredential` proof fields
  from #39 also ship in 0.2.0; neither change adds an RPC or further breakage.

No compatibility aliases, generated tonic adapters, or dual production
protocol are provided. Heddle and Weft must consume the same 0.2 revision.
